# Vercel 배포 가이드

이 프로젝트를 Vercel에 배포하는 두 가지 방법을 설명합니다.

## 1. Vercel CLI 배포 (현재 상황에 추천)
현재 로컬에 Git 저장소가 없거나, GitHub를 거치지 않고 내 컴퓨터에서 바로 배포하고 싶을 때 사용합니다.

**명령어:**
```bash
# 터미널에서 실행
npx vercel --prod
```

**과정:**
1. 위 명령어를 입력합니다.
2. 로그인 또는 설정 확인 질문이 나오면 `Enter` 키를 눌러 기본값으로 진행합니다.
3. 업로드가 완료되면 배포된 URL이 출력됩니다.
4. 해당 URL에서 변경사항을 바로 확인할 수 있습니다.

**특징:**
- **장점:** 설정이 간편하고 배포 속도가 빠릅니다.
- **단점:** 코드가 GitHub에 백업되지 않습니다.

---

## 2. Git 기반 배포 (권장 표준 방식)
프로젝트 코드를 GitHub에 저장하면 Vercel이 자동으로 감지하여 배포하는 방식입니다.

**워크플로우:**
1. **GitHub에 코드 저장 (Push)**
   ```bash
   git add .
   git commit -m "수정 내용 설명"
   git push origin main
   ```
2. **Vercel 자동 감지**
   - GitHub에 코드가 올라가는 순간 Vercel이 이를 감지하고 배포를 시작합니다.
3. **배포 완료**
   - 잠시 후 사이트가 자동으로 업데이트됩니다.

**특징:**
- **장점:** 코드가 안전하게 백업되고 수정 이력이 남습니다. 협업에 필수적입니다.
- **단점:** 처음에 Git 저장소 연결 설정이 필요합니다.

---

## 요약
- **빠른 수정 반영:** `npx vercel --prod`
- **정석적인 관리:** Git 설정 후 `git push`
